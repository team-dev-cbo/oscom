Déclenchement

 

Un déclenchement est définie par :

 

Nom du déclenchement

Ce nom peut être composé de lettres et de chiffres ainsi que des espaces mais pas d'accent.

 

Description

Un commentaire permet de décrire le rôle et les fonctionnalités du déclenchement.

 

Chaîne de déclenchement

Un scénario s'exécute à la réception d'un message particulier provenant d'un utilisateur (qui peut être un terminal d'atelier) autorisé à déclencher le scénario.

Le message de déclenchement du scénario peut être constitué de tout ou partie (début) du message reçu.

Par exemple : le déclenchement par "a" indique que l'appui sur F1 (= a) ou que la réception d'un message commençant par "a" ("aaaahf" par exemple) active ce scénario.

Remarque : si le message reçu correspond à plusieurs déclenchements possibles, c'est le message de déclenchement le plus long qui est seul pris en compte.

Cas particulier : si le message de déclenchement est vide, ce scénario se déclenche si aucun autre scénario ne peut être activé par le message reçu (scénario par défaut).

 

Liste des périphériques du déclenchement

Un scénario ne s'exécute que si le message reçu provient d'un des périphériques spécifiés dans la liste : au moins un périphérique doit obligatoirement être sélectionné.

Exemple : réception d'un message provenant du clavier ou d'un code à barre saisi sur terminal d'atelier.

 

Utilisateurs autorisés

Noms des utilisateurs autorisés à déclencher le scénario.

 

Scénario associé

Nom : Nom du scénario (lpx ou graphique) associé au déclenchement

Paramètres : Parmètres du scénario

Périphériques utilisés : Périphérique utilisé par le scénario
