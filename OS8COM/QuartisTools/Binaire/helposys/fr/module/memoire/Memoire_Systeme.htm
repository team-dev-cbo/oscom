Mémoire Système

 

Une mémoire systéme est définie comme une mémoire mais :

 

L'utilisateur ne peut ni créer, ni supprimer, ni modifier une mémoire système, ni lire le contenu de la mémoire système via l'interface graphique.

Le contenu de la mémoire système est lu via un scénario.

Seule la valeur des mémoires "Code Scénario", "Chemin QuartisMsg"et "Profiling" peuvent être modifiées par un scénario.

Ex : La méthode EcritMemoire  pour un scénario LPX.

 

 

Description de chaque mémoire système :

 

Nom de la mémoire système

Description

Chemin QuartisMsg

Chemin d'accès à la base de données Quartis Messenger

Code Scénario

Code du scénario.

Date action

Date du message reçu par le scénario en cours.

Date début

Date de début du lancement du scénario en cours.

Date/Heure action

Date/Heure du message reçu par le scénario en cours.

Date/Heure début

Date/Heure de début du lancement du scénario en cours.

Heure action

Heure du message reçu par le scénario en cours.

Heure début

Heure de début du lancement du scénario en cours.

Id Utilisateur

Numéro ID de l'utilisateur qui a déclenché le scénario en cours.

Message action

Message reçu par le scénario en cours.

Message déclenchement

Message de déclenchement.

Nom Scénario

Nom du scénario en cours d'exécution.

Nom du serveur

Nom de la machine du serveur.

Nom Utilisateur

Nom de l'utilisateur qui a déclenché le scénario en cours.

Paramètre du scénario

Liste des paramètres du scénario en cours.

Périphérique action

Périphérique du message reçu.

Périphérique déclenchement

Périphérique qui a permis le déclenchement du message.

Port Tomcat

Numéro de port de Tomcat de QuartisTools

Profiling

Si la valeur de la mémoire (=X) est supérieur à zéro, active le profiler et  les données seront conservées X heures par le profiler.

Répertoire application

Répertoire de l'application.

Répertoire Tomcat

Chemin du répertoire webapps de Tomcat de QuartisTools

Répertoire web

Répertoire de base du serveur web.

Terminal

Numéro du terminal.

Type Utilisateur

Type de l'utilisateur connecté (Interface, Console, Driver Scrut, Driver Etat, ...)
