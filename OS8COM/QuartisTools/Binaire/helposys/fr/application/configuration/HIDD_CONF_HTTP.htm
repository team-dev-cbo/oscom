Configuration du serveur web

Menu "Application/Configuration"--Commande "Serveur Web"

 

Numéro port OSYSHTTP :

Le port standard TCP/IP utilisé pour le serveur web est 80. Si ce port est déjà utilisé pour une autre application, il est nécessaire de lui donner une autre valeur.

Exemple : Une même machine ( ordinateur ) peut avoir plusieurs applications de type serveur Web.

 

Répertoire de base :

Répertoire de base de l'application contenant tous les fichiers htm ou html utile au déroulement de l'application. Ce répertoire se trouve sur le disque dur du serveur. Ce répertoire peut être lui-même divisé en sous-répertoire pour classer les différentes sortes de fichier : image, vidéo, son, etc...

 

Durée maximum d'attente en lecture sur une connexion :

Pour une connexion, durée (en secondes) maximum d'attente d'une nouvelle demande provenant d'un navigateur.

 

Temps maximum de lecture pour une connexion :

   Temps maximum qu'une connexion a pour lire des informations supplémentaires provenant du navigateur.  (Ex : Si la taille du formulaire reçue est inférieure à la taille totale du formulaire, lecture sur la connexion du reste du formulaire).

 

Nom du fichier par défaut :

   Si aucun nom n'est spécifié, le nom du fichier par défaut sera default.htm.

 

Taille du fichier temporaire :

   Taille maximum (en Kilo octets) que peut avoir le fichier temporaire.

 

Nombre maximum de connexions simultanées :

   Nombre maximum de connexions que le serveur web peut avoir.

 

Envoi de la clé au serveur web :

   Si oui, le serveur web renvoie au navigateur son nom et sa version.

   Sinon, le serveur web ne renvoie pas son nom et sa version.

 

Autorisation CGI :

Les scripts CGI ( programmes externes ) peuvent être autorisés ou non.

Le bouton permet de définir la liste des fichiers associés à un script CGI.

( voir Liste des extensions exécutables)
